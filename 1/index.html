Hello, World! One important thing to know about HTML is that it is does not preserve whitespace. What does that mean? Open this file in your browser and you'll notice that it's all one big paragraph. Often, that's not what you want. HTML treats any sequence of 'spaces' (known as whitespace) as if it were just one space character. Whitespace also includes new lines and tabs.

So if you want to create a new paragraph, you have to do this.

The 'p' stands for paragraph. This tells the browser to create a paragraph element. What's an element? That's a difficult question, so I wouldn't worry about it for now.

One great thing about the way HTML treats whitespace is that you can lay out your HTML however you like, and you'll still get the same result. You will notice that I'm breaking up my paragraphs into lines of around 80 characters to keep my source code readable. But the browser will still use all the space it has available.

One of the most important things to keep in mind when writing code is to make it easy to read. It takes a while to learn how to do this, but keep it in mind.

When you come across a tag you don't know, you can look it up. One of the best references for HTML is Mozilla Developer Network. It's by the same people that make Firefox. Here is a list of HTML elements.

A lot of it might not make sense to you immediately, but it's still worth reading, for the same reason you might read something you don't understand when learning a new language. It gets you accustomed to the language. You can always google terms you don't know to see if you can figure them out.

You should experiment. Change anything you like, try to break it and see what happens. It's actually very important to break things, because even the most experienced developers break things many times each day. If you can recognise what a broken thing looks like, you'll be better prepared when you encounter it unexpectedly.
